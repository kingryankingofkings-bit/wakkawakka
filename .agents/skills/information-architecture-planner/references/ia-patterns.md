# Information Architecture Patterns

## Common Patterns

### Hierarchy

Tree structure with clear parent-child relationships. Best for: corporate sites, documentation.

### Sequential (Wizard)

Linear progression through steps. Best for: checkout, onboarding, forms.

### Matrix

Users choose their own path via cross-linking. Best for: content-heavy sites, wikis.

### Database

Content organized by facets (filter, sort, search). Best for: ecommerce, directories, listings.

## URL Best Practices

- Use kebab-case: `/about-our-company`
- No trailing slashes inconsistency
- No file extensions in URLs
- Keep under 75 characters
- Use canonical URLs for duplicate content
